## 1. SEO란 무엇인가

**SEO = Search Engine Optimization, 검색엔진 최적화**

> 쉬운 비유: 도서관 사서(검색엔진)가 "이 책이 어떤 내용인지" 빠르게 파악할 수 있도록 책의 제목·목차·색인을 잘 정리해두는 것.

검색엔진(구글, 네이버)은 사용자가 단어를 입력하면 수백억 개의 웹페이지 중 "가장 관련 있고 믿을 만한" 페이지를 순서대로 보여준다.
SEO는 내 페이지가 이 순위에서 앞에 오도록 최적화하는 작업이다.

### 왜 중요한가?
- 검색 1위 클릭률 ≈ 27%, 2위 ≈ 15%, 3위 ≈ 10%
- 10위(1페이지 마지막) 이후는 클릭률 1% 미만
- 광고비 없이 꾸준히 방문자를 유입시킬 수 있는 유일한 방법

### SEO의 3축
| 축 | 내용 |
|---|---|
| 기술적 SEO | 검색엔진이 페이지를 빠르게 읽고 색인할 수 있게 |
| 온페이지 SEO | 콘텐츠·제목·메타태그를 사용자 의도에 맞게 |
| 오프페이지 SEO | 다른 사이트로부터 링크(신뢰도)를 얻는 것 |

## 2. 검색엔진은 어떻게 작동하는가

### 3단계 프로세스

```
크롤링(Crawling) → 색인(Indexing) → 랭킹(Ranking)
```

#### 크롤링
- 구글봇(Googlebot)이라는 로봇이 인터넷을 돌아다니며 페이지를 수집
- 링크를 타고 이동 → 새 페이지 발견 → 수집
- `robots.txt`로 "여기는 오지 마" 지시 가능

#### 색인 (Index)
- 수집한 페이지의 내용을 분석해서 구글의 거대한 데이터베이스에 저장
- **색인이 안 되면 검색 결과에 아예 안 나온다**
- `site:내도메인.com` 으로 색인 여부 확인 가능

#### 랭킹
- 사용자가 검색어를 입력하면 200개 이상의 요소를 고려해 순위 결정
- 주요 요소: 콘텐츠 관련성, 페이지 속도, 모바일 친화성, 백링크 수와 질

## 3. 온페이지 SEO

> 내 HTML 파일 안에서 최적화할 수 있는 모든 것

### `<title>` 태그 — 가장 중요한 단 한 줄

```html
<title>서울 강남 헬스장 추천 TOP 5 | 가격·위치 비교 2025</title>
```

- 검색 결과 파란 제목으로 표시된다
- **50~60자** 이내 (초과하면 잘림)
- 타겟 키워드를 앞쪽에 배치
- 페이지마다 고유하게 작성

### `<meta name="description">` — 검색 결과 요약문

```html
<meta name="description" content="강남 헬스장 5곳의 월 회비, 위치, 시설을 직접 방문해 비교했습니다. 2025년 최신 가격 포함.">
```

- 검색 결과에서 제목 아래 회색 글씨로 표시
- **150~160자** 이내
- 클릭률(CTR)에 직접 영향 — 순위는 올리지 않지만 클릭을 유도
- 키워드를 자연스럽게 포함

### `<h1>` ~ `<h6>` — 문서 구조

```html
<h1>강남 헬스장 추천 (1개만 사용)</h1>
  <h2>가격 비교</h2>
    <h3>강남구청역 헬스장</h3>
  <h2>시설 비교</h2>
```

- `<h1>`은 페이지당 **단 1개** — 이 페이지의 핵심 주제
- 목차처럼 계층 구조로 작성
- 검색엔진이 페이지 구조를 파악하는 데 사용

### URL 구조

```
좋음: /blog/gangnam-gym-top5
나쁨: /page?id=1234&cat=ab
```

- 짧고, 의미 있는 단어 사용
- 한글보다 영문 소문자 + 하이픈(-) 권장 (인코딩 문제 방지)

### 이미지 최적화

```html
<img src="gym-interior.webp" alt="강남 A헬스장 내부 전경 — 유산소 기구 50대">
```

- `alt` 속성 필수 — 크롤러는 이미지를 "보지" 못하고 alt 텍스트를 읽는다
- WebP 포맷으로 용량 절감 (JPEG 대비 30% 가벼움)
- 파일명도 의미 있게 (gym-interior.jpg O, IMG_20240101.jpg X)

## 4. 기술적 SEO

### 시맨틱 HTML — "이게 무슨 영역인지" 태그로 알려주기

```html
<!-- 나쁜 예: 의미 없는 div 덩어리 -->
<div class="header">...</div>
<div class="main">...</div>

<!-- 좋은 예: 시맨틱 태그 -->
<header>...</header>
<nav>...</nav>
<main>
  <article>...</article>
  <aside>...</aside>
</main>
<footer>...</footer>
```

| 태그 | 의미 |
|------|------|
| `<header>` | 페이지/섹션 상단 (로고, 네비게이션) |
| `<nav>` | 내비게이션 메뉴 |
| `<main>` | 페이지 핵심 콘텐츠 (단 1개) |
| `<article>` | 독립적으로 배포 가능한 콘텐츠 (블로그 글, 뉴스) |
| `<section>` | 주제별로 묶인 콘텐츠 단위 |
| `<aside>` | 부가 정보 (사이드바, 광고) |
| `<footer>` | 페이지 하단 |

### Open Graph — SNS 공유 시 미리보기 카드

```html
<meta property="og:title" content="강남 헬스장 추천 TOP 5">
<meta property="og:description" content="직접 방문해 비교한 가격과 시설">
<meta property="og:image" content="https://내도메인.com/og-image.jpg">
<meta property="og:url" content="https://내도메인.com/gangnam-gym">
<meta property="og:type" content="article">
```

- 카카오톡, 페이스북, 트위터 등 공유 시 이미지+제목+설명이 자동 표시
- 이미지 권장 크기: **1200 × 630px**

### `robots.txt` — 크롤러에게 규칙 알려주기

```
User-agent: *
Allow: /
Disallow: /admin/
Sitemap: https://내도메인.com/sitemap.xml
```

- 루트 디렉터리(`/`)에 위치
- 크롤링 허용/차단 규칙 설정
- `Sitemap` 경로를 명시하면 크롤러가 발견하기 쉬워짐

### `sitemap.xml` — 모든 페이지 목록을 검색엔진에 제출

```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://내도메인.com/</loc>
    <lastmod>2025-05-01</lastmod>
    <priority>1.0</priority>
  </url>
</urlset>
```

- 새 콘텐츠를 빠르게 색인시키는 가장 직접적인 방법
- Google Search Console에 등록하면 수동 제출 가능

### HTTPS — 보안 연결

- `http://` → `https://` : 구글 공식 순위 요소
- SSL 인증서 없으면 브라우저에 "안전하지 않음" 표시 → 이탈률 급증
- 무료 인증서: Let's Encrypt

### 모바일 최적화 (Mobile-First Indexing)

```html
<meta name="viewport" content="width=device-width, initial-scale=1">
```

- 구글은 **모바일 버전**을 기준으로 순위를 매긴다 (2024년부터 완전 전환)
- 반응형 디자인(`@media`) 필수

## 5. Core Web Vitals

> 구글이 2021년부터 공식 순위 요소로 채택한 3가지 성능 지표

### LCP (Largest Contentful Paint) — 주요 콘텐츠 로딩 속도

- 페이지에서 **가장 큰 이미지나 텍스트 블록**이 화면에 나타나는 시간
- 목표: **2.5초 이내**
- 개선법:
  - 이미지에 `loading="eager"` + `fetchpriority="high"` 속성 (첫 화면 이미지)
  - 이미지 WebP 변환 + 크기 최적화
  - 서버 응답 속도 개선 (TTFB)

### CLS (Cumulative Layout Shift) — 레이아웃 흔들림

- 페이지 로딩 중 요소가 **갑자기 이동**하는 정도
- 목표: **0.1 이하** (0에 가까울수록 좋음)
- 원인: 이미지/광고에 `width`·`height` 미지정, 늦게 로드되는 폰트
- 개선법:
  ```html
  <!-- 이미지에 반드시 크기 지정 -->
  <img src="photo.webp" width="800" height="450" alt="...">
  ```

### INP (Interaction to Next Paint) — 클릭 반응 속도

- 사용자가 클릭/탭/키입력 후 화면이 **반응하는 시간**
- 목표: **200ms 이내**
- 개선법: 무거운 JavaScript 분리, 긴 작업(Long Task) 분할

### 측정 방법
1. [PageSpeed Insights](https://pagespeed.web.dev/) — URL 입력 즉시 측정
2. Chrome DevTools → Lighthouse 탭
3. Google Search Console → Core Web Vitals 리포트

## 6. 구조화 데이터 (JSON-LD)

> 검색 결과에 별점, 가격, FAQ 등을 추가로 표시하게 만드는 코드

### 리치 스니펫이란?
일반 검색 결과는 제목+URL+설명 3줄이지만, 구조화 데이터가 있으면:
- ⭐⭐⭐⭐☆ 4.2 (리뷰 208개)
- 가격: ₩15,000
- 재고: 있음

이런 추가 정보가 표시된다.

### 예시: 기사(Article) 타입

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "강남 헬스장 추천 TOP 5",
  "author": {
    "@type": "Person",
    "name": "홍길동"
  },
  "datePublished": "2025-05-01",
  "dateModified": "2025-05-07",
  "image": "https://내도메인.com/gym-photo.webp"
}
</script>
```

### 예시: FAQ 타입 (검색 결과에 아코디언 표시)

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "강남 헬스장 월 회비는 얼마인가요?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "강남구 헬스장 평균 월 회비는 6~12만원입니다."
      }
    }
  ]
}
</script>
```

### 주요 Schema.org 타입
| 타입 | 사용 시 |
|------|---------|
| `Article` | 블로그 글, 뉴스 |
| `FAQPage` | FAQ 페이지 |
| `Product` | 상품 페이지 |
| `LocalBusiness` | 지역 업체 |
| `BreadcrumbList` | 상단 경로(홈 > 카테고리 > 글) |
| `WebSite` | 홈페이지 |

## 7. 네이버 SEO

> 구글과 알고리즘이 달라서 별도 전략이 필요하다

### 구글 vs 네이버 핵심 차이

| 항목 | 구글 | 네이버 |
|------|------|--------|
| 색인 방식 | 모든 외부 웹 크롤링 | 자사 서비스 우선 (블로그, 카페, 지식인) |
| 순위 핵심 | 백링크 + 콘텐츠 질 | 콘텐츠 내 키워드 밀도 + 최신성 |
| 개인 블로그 | 어렵다 | 네이버 블로그면 유리 |
| 도구 | Search Console | 네이버 서치어드바이저 |

### 네이버 웹마스터 도구 (서치어드바이저)

1. [searchadvisor.naver.com](https://searchadvisor.naver.com/) 에서 사이트 등록
2. 소유권 확인 파일(`naver_*.html`) 루트에 업로드
3. sitemap.xml 제출
4. 수집 요청 (크롤링 촉진)

### 네이버 SEO 특화 메타태그

```html
<!-- 네이버 서치어드바이저 인증 -->
<meta name="naver-site-verification" content="인증코드">
```

### 네이버 검색 알고리즘 특징
- **C-Rank**: 콘텐츠 생산자의 신뢰도 점수 (꾸준히 양질의 글 작성)
- **D.I.A.**: 문서의 정보성 점수 — 직접 경험, 구체적 수치 포함 여부
- 네이버 블로그 > 네이버 카페 > 일반 웹사이트 순으로 상단 노출 유리
- 외부 사이트라면 **네이버 구조화 데이터** + 사이트맵 제출이 핵심

## 8. 백링크

> 다른 사이트가 내 사이트 링크를 걸어주는 것 = 신뢰 투표

### 왜 중요한가?
구글은 "신뢰할 수 있는 사이트가 링크를 걸어준 페이지"를 높이 평가한다.
뉴욕타임스가 내 블로그 링크를 걸면 점수가 크게 오른다.

### 좋은 백링크 vs 나쁜 백링크

| 좋은 백링크 | 나쁜 백링크 |
|------------|------------|
| 주제 관련 사이트에서 | 주제 무관 스팸 사이트 |
| 자연스럽게 언급 | 링크 구매/교환 |
| 도메인 권위(DA) 높은 사이트 | 신생 저품질 사이트 |

### 백링크를 얻는 현실적 방법
1. **좋은 콘텐츠 만들기** — 자연스럽게 인용되는 것이 최선
2. 게스트 포스팅 — 다른 블로그에 기고
3. 커뮤니티 참여 — 해당 주제 포럼에서 전문성 보여주기
4. 공개 데이터/도구 제공 — 인용 가능한 자료 만들기

## 9. 검색 의도 (Search Intent)

> 제목에 키워드가 있어도 의도가 맞지 않으면 순위가 안 오른다

### 4가지 검색 의도

| 의도 | 예시 검색어 | 원하는 결과 |
|------|------------|------------|
| 정보형 | "강남 헬스장 가격" | 가격 비교 정보 |
| 탐색형 | "강남 A헬스장" | 그 업체의 공식 사이트 |
| 거래형 | "헬스장 1개월 등록" | 결제/예약 페이지 |
| 상업형 | "헬스장 추천" | 비교/리뷰 페이지 |

### 실전 적용
- "헬스장 추천" 검색어 → 상업형 → 비교 글이어야 순위 오름
- 제품 설명 페이지로 만들면 의도 불일치 → 순위 안 오름

## 10. 측정 도구

### Google Search Console (무료)
- 내 사이트가 구글에서 어떤 키워드로 노출되는지 확인
- 클릭수, 노출수, 평균 순위 확인
- 색인 오류 확인 및 재색인 요청
- sitemap.xml 제출

### Google Analytics 4 (무료)
- 방문자 수, 체류 시간, 이탈률 측정
- 어떤 페이지가 많이 보이는지 파악

### PageSpeed Insights (무료)
- Core Web Vitals 점수 측정
- 개선 항목과 방법 제시

## 면접 단골 질문

**Q. SEO를 위해 실제로 어떤 작업을 했나요?**
> A. 시맨틱 HTML 구조 적용, 메타 태그(`<title>`, `<meta description>`) 최적화, Core Web Vitals 측정 후 이미지 WebP 변환과 크기 명시로 CLS 해결, JSON-LD로 구조화 데이터 추가, sitemap.xml과 robots.txt 작성 후 Search Console에 제출했습니다.

**Q. 구글과 네이버 SEO 차이는?**
> A. 구글은 백링크와 콘텐츠 질을 중심으로 평가하고 전 세계 웹을 크롤링합니다. 네이버는 자사 서비스(블로그, 카페)를 우선 노출하고, 콘텐츠 내 키워드 밀도와 최신성을 더 중요시합니다. 외부 사이트의 경우 네이버 서치어드바이저에 사이트맵 등록이 필수입니다.

**Q. Core Web Vitals가 무엇인가요?**
> A. 구글이 사용자 경험을 정량화한 3가지 지표입니다. LCP(주요 콘텐츠 로딩 시간, 목표 2.5초 이내), CLS(레이아웃 흔들림, 목표 0.1 이하), INP(클릭 반응 속도, 목표 200ms 이내)입니다. 2021년부터 공식 순위 요소로 사용됩니다.

**Q. 구조화 데이터란?**
> A. Schema.org 표준을 따르는 JSON-LD 코드로, 검색 결과에 별점·가격·FAQ 같은 부가 정보(리치 스니펫)를 표시하게 합니다. 클릭률 향상에 직접 기여합니다.
