# SEO 블로그 최적화 스킬 (Lite)

블로그 플랫폼에 상관없이 검색 최적화된 글을 작성하거나, 기존 글의 SEO를 점검·개선하는 범용 스킬이다.

---

## 0. 키워드 확인

글을 작성하거나 점검하기 전에 키워드를 먼저 확정한다.

### 키워드가 지정된 경우

사용자가 제공한 키워드를 그대로 핵심 키워드로 사용한다. 필요하면 관련 키워드(LSI 키워드)를 보충해 글에 자연스럽게 배치한다.

### 주제만 지정된 경우

아래 형식으로 키워드 후보를 먼저 제안하고, 사용자 확인을 받은 뒤 진행한다.

```
## 키워드 제안

- **핵심 키워드**: [1개]
- **관련 키워드**: [3~5개]
- **선정 근거**: [왜 이 키워드가 검색 수요가 있는지 1~2문장]

→ 이 키워드로 진행할까요? 수정이 필요하면 말씀해 주세요.
```

### 기존 글 점검 시

사용자가 기존 글을 제공한 경우, 본문에서 핵심 키워드를 추출한 뒤 사용자에게 확인받는다. 사용자가 노리는 키워드가 따로 있을 수 있으므로 반드시 확인한다.

---

## 1. SEO 글 구조

검색엔진이 좋아하는 글 구조의 원칙이다. 새 글 작성 시 이 구조를 따르고, 기존 글 점검 시 이 구조에 맞는지 진단한다.

### 1-1. 도입부

첫 단락에서 핵심 키워드가 자연스럽게 등장해야 한다. 검색엔진은 글의 초반부에서 주제 관련성을 판단한다.

### 1-2. 본문

글 전체 분량은 **1,500~3,000자**, **H2 섹션 3~5개**를 권장한다.

- **H2와 H3를 적절히 사용**해서 글을 구조화한다. 검색엔진은 헤딩 태그로 글의 위계와 주제를 파악한다.
- **소제목에 키워드를 포함**한다. 핵심 키워드와 관련 키워드가 소제목에 자연스럽게 녹아야 한다.
- **정의 문단 1개 이상**: 글의 핵심 개념을 "[용어]란 [정의]이다" 형식으로 설명하는 독립 문단을 둔다. 이 문단은 AI 검색엔진이 답변으로 인용(Featured Snippet)하기 좋은 구조다.
- **각 H2 섹션 본문은 300자 이상**으로 작성한다. 너무 짧은 섹션은 검색엔진에서 얇은 콘텐츠(thin content)로 평가될 수 있다.

### 1-3. FAQ 섹션 (권장)

글 하단에 Q&A 3~5개를 배치한다. 독자가 실제로 검색할 법한 질문을 사용한다.

```
## 자주 묻는 질문

**Q1. [질문]**
A. [답변 3~5줄]

**Q2. [질문]**
A. [답변 3~5줄]
```

FAQ는 구글 검색 결과에서 풍부한 스니펫(Rich Snippet)으로 노출될 수 있어 클릭률 향상에 도움이 된다.

---

## 2. SEO 핵심 규칙

### 2-1. 키워드 배치

- **첫 100자 안에 핵심 키워드 2회 이상** 자연스럽게 배치한다.
- **소제목(H2/H3)에 키워드를 포함**한다.
- 키워드를 억지로 반복하지 않되, 글 전체에 걸쳐 자연스럽게 분포시킨다.
- 동의어와 관련 키워드(LSI 키워드)를 함께 사용해 주제 커버리지를 넓힌다.

### 2-2. 제목 작성

- **핵심 키워드를 제목 앞쪽에** 배치한다.
- **25~35자 이내**로 유지한다. 검색 결과에서 잘리지 않는 길이다.
- **숫자를 포함**하면 클릭률이 올라간다 ("5가지 방법", "3단계 가이드").
- 제목과 본문 내용이 일치해야 한다. 클릭베이트는 이탈률을 높여 SEO에 악영향을 준다.

### 2-3. 메타 설명 (SEO Description)

글 작성·점검 완료 후 메타 설명 초안을 함께 출력한다:

- **140~156자** 사이로 작성한다. 구글 검색 결과에서 잘리지 않는 길이다.
- 첫 문장에 핵심 키워드를 넣는다.
- 구체적 숫자나 혜택을 포함해 클릭을 유도한다.
- 본문 도입부를 그대로 복사하지 않는다. 메타 설명은 검색 결과에서 독자가 클릭 여부를 결정하는 별도의 텍스트다.

### 2-4. URL 슬러그

- 영문 소문자, 하이픈으로 단어를 구분한다.
- 핵심 키워드를 앞쪽에 배치한다.
- 관사(the, a)는 생략한다.
- 30~50자 사이가 적당하다.

## 3. 출력 형식

### 3-1. 새 글 작성 시

마크다운 형식으로 출력한다:

```markdown
# [제목]

[도입부]

## [H2 소제목 - 키워드 포함]

[본문 300자 이상]

### [H3 소제목 - 필요시]

[세부 내용]

## 자주 묻는 질문

**Q1. [질문]**
A. [답변]

[마무리]
```

### 3-2. 기존 글 점검 시

아래 형식으로 SEO 진단 리포트를 출력한다:

```
## SEO 진단 리포트

### 종합 점수: X / 10

### 잘 된 점
- [구체적 항목]

### 개선 필요 항목
1. **[항목명]**: [현재 상태] → [권장 사항]
2. **[항목명]**: [현재 상태] → [권장 사항]

### 수정 제안
> [구체적인 수정 예시를 포함한 개선안]
```

### 3-3. SEO 세팅 출력

본문 또는 진단 리포트 아래에 SEO 세팅 초안을 출력한다:

```
---

## SEO 세팅 초안

### 제목
- 권장: [제목] (XX자)
- 대안: [제목]

### URL 슬러그
- 권장: `[slug]`
- 대안: `[slug]`

### SEO Description
> [140~156자 완성 텍스트]

### 추천 태그/카테고리
1. [태그]
2. [태그]
3. [태그]
4. [태그]
5. [태그]
```

---

## 4. SEO 체크리스트

글 작성 또는 점검 완료 후 아래 항목을 자동으로 검증한다. 미달 항목이 있으면 개선안을 제시한다.

### 키워드
- [ ] 핵심 키워드가 확정되었는가?
- [ ] 첫 100자 안에 핵심 키워드가 2회 이상 등장하는가?
- [ ] 소제목(H2/H3)에 키워드가 포함되어 있는가?
- [ ] LSI 키워드(동의어/관련어)가 글 전체에 분포되어 있는가?

### 구조
- [ ] H2 섹션이 3~5개인가?
- [ ] H2/H3 위계가 올바른가? (H3가 H2 없이 단독 사용되지 않는가?)
- [ ] 각 H2 섹션 본문이 300자 이상인가?
- [ ] 글 전체 분량이 1,500~3,000자인가?
- [ ] 정의 문단("[용어]란 [정의]이다")이 1개 이상 있는가?
- [ ] FAQ 섹션이 포함되어 있는가?

### 메타 정보
- [ ] 제목이 25~35자 이내인가?
- [ ] 제목 앞쪽에 핵심 키워드가 배치되었는가?
- [ ] SEO Description이 140~156자인가?
- [ ] SEO Description 첫 문장에 핵심 키워드가 있는가?
- [ ] URL 슬러그가 영문 소문자+하이픈 형식인가?
- [ ] URL 슬러그에 핵심 키워드가 포함되어 있는가?
